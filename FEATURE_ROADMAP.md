# Storage Manager Feature Status

자동 삭제 없이 읽기 전용 모니터링과 안전한 야간 수집을 우선합니다.

## 이번 반영 완료

1. **inode 사용률**
   `df -Pi`를 byte 사용률과 함께 수집하고 90% 주의, 95% 경고를 적용합니다.
2. **Full 예상일**
   최근 7일·30일 기울기로 경고 기준과 100% 도달 예상일을 표시합니다. 표본 부족,
   감소·정체 추세, 10년 초과 예상은 `예측 불가`로 처리합니다.
3. **증가 이상 탐지**
   최근 양의 증가량 중앙값·MAD, 3배 배수, 최소 100GB 조건을 함께 사용합니다.
4. **수집 신선도 감시**
   계정별 마지막 nightly 수집 경과시간을 표시하고 07시 건강 점검 cron이 누락·실패를
   GUI 없이 알립니다.
5. **Quota·공유 filesystem·내부 알림**
   quota JSON command, filesystem별 계정 그룹, outbox/command/webhook 어댑터와
   cooldown을 제공합니다.
6. **정리 검토 후보**
   기본 100GB 이상, 30일 관찰, 최근 30일 활동 없음 조건의 보고서를 생성합니다.
   실제 삭제 명령은 실행하지 않습니다.
7. **15분 급증 감시와 MATE 로컬 알림**
   15분 `df` 표본으로 100GB 급증과 6시간·2시간 이내 FULL을 감지하고, GUI가
   닫혀 있어도 로그인 세션의 독립 트레이 notifier가 outbox 경고를 표시합니다.
8. **전역 데이터 위치와 런타임 진단**
   quota가 제한된 개인 계정에는 1KB 미만 위치 포인터만 두고 실제 데이터는 최초
   선택한 여유 있는 경로에 저장합니다. `run.csh --diagnose`가 Python·JSON·SQLite·
   PyQt5·그룹·쓰기 권한과 현재 관리 데이터 크기를 한 번에 확인합니다.
9. **관리자 전용 경로 검색**
   고정 PIN으로 현재 세션에서만 검색 탭을 노출하고, 계정별 opt-in 방식으로 파일·
   디렉터리 이름과 확장자를 별도 SQLite에 인덱싱합니다. 디렉터리 체크포인트로
   재개하며 일일 변경 스트림 재사용과 7일 전체 대조를 함께 사용합니다.
10. **간결한 운영 UI**
   계정명·경로 양방향 자동 완성, inode 도움말, 상태형 cron/scan/notifier 버튼,
   넓은 최근 수집 열, X 종료 시 cron·트레이 지속 안내를 제공합니다.

## 요청에 따라 제외

- 담당자, 과제명, due date, 작업/백업 계정 연결과 같은 계정 생명주기 메타데이터는
  이번 범위에서 제외했습니다.

## 실제 VWP 확인 후 고려

1. 파일시스템이 snapshot diff 또는 changelog API를 제공하면 전체 `find` 순회를
   대체할 수 있습니다. 30~50TB 환경에서 가장 큰 성능 개선 후보입니다.
2. 여러 관리자가 하나의 이력을 공유해야 하면 네트워크 SQLite 대신 중앙 수집기와
   읽기 전용 GUI/API 구조가 적합합니다.
3. 역할 기반 권한과 계정 편집·cron 변경·중지 요청 감사 로그를 추가할 수 있습니다.
4. 계획된 신규 데이터량을 가상으로 더하는 30/60/90일 용량 시나리오를 추가할 수
   있습니다.
